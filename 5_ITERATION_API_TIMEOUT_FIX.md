# Итерация 5: Увеличение таймаута для запросов к LiveScore API

## Проблема
При запросах к LiveScore API возникали таймауты:
```
Error [TimeoutError]: The operation was aborted due to timeout
at async LoggedFetch.makeApiRequest (src/lib/http/livescore/logged-fetch.ts:274:22)
```

API не успевал ответить за 10 секунд, что приводило к ошибкам при загрузке:
- Матчей (matches/live.json)
- Фикстур (fixtures/matches.json)
- Статистики матчей (matches/stats.json)
- Событий матчей (matches/events.json)

## Причина
LiveScore API иногда работает медленно и требует больше времени для ответа. Таймаут в 10 секунд был недостаточным.

## Решение
Увеличен таймаут с 10 секунд на 30 секунд в файле `src/lib/http/livescore/logged-fetch.ts`:

```typescript
// Было:
signal: AbortSignal.timeout(10000),

// Стало:
signal: AbortSignal.timeout(30000),
```

## Файлы, затронутые изменением
- `src/lib/http/livescore/logged-fetch.ts` - увеличен таймаут в методе `makeApiRequest`

## Статус
✅ Завершено. Таймаут увеличен до 30 секунд для более надежной работы с медленным API.

## Результат
- Запросы к LiveScore API теперь имеют достаточно времени для ответа
- Снижается вероятность таймаутов при нормальной работе API
- Логирование ошибок остается активным для отслеживания проблем

## Примечание
Если таймауты продолжат возникать, может потребоваться:
1. Дальнейшее увеличение таймаута
2. Реализация retry логики с экспоненциальной задержкой
3. Использование кэша для снижения количества запросов к API
