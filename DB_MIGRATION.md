# Инструкция по миграции базы данных

Этот документ описывает, как перенести вашу локальную базу данных на удаленный сервер с помощью двух скриптов.

## Шаг 1: Экспорт и загрузка (локальный компьютер)

Запустите эту команду на **вашем локальном компьютере** из корневой папки проекта. Она создаст дамп вашей локальной базы `payload` и загрузит его на сервер.

```bash
pnpm run db:push-to-prod
```

Скрипт запросит у вас пароль для SSH-доступа к серверу.

## Шаг 2: Импорт и очистка (удаленный сервер)

После успешного выполнения предыдущего шага, подключитесь к **вашему удаленному серверу** по SSH, перейдите в папку с проектом и запустите второй скрипт.

```bash
ssh root@<IP_адрес_сервера>
cd ro-ko-football
chmod +x import-and-cleanup.sh && ./import-and-cleanup.sh
```

Вы также можете использовать команду `pnpm run db:apply-on-prod` на локальной машине, чтобы получить напоминание о том, какую команду выполнить на сервере.

Этот скрипт импортирует данные в вашу базу данных на сервере и удалит временные файлы дампа.

---

✅ **Готово!** Ваша база данных перенесена.
