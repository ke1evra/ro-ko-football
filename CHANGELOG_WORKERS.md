# 📝 Changelog: Оптимизация воркеров и добавление расчета статистики

## Дата: 17.12.2024

### ✨ Новые возможности

#### 1. Автоматический расчет статистики прогнозов

- Добавлен воркер для автоматического расчета статистики прогнозов
- Работает в фоновом режиме с интервалом 10 минут
- Автоматически обрабатывает завершенные матчи

#### 2. Оптимизация Docker для воркеров

- Создан легкий `Dockerfile.worker` специально для воркеров
- Время сборки сокращено с 10-15 минут до 2-3 минут
- Размер образа уменьшен с ~500MB до ~200MB
- Добавлен `.dockerignore` для исключения ненужных файлов

### 🚀 Использование

#### Запуск на продакшене

```bash
# Импорт матчей
docker compose -f docker-compose.matches-import-forward.yml up -d

# Расчет статистики
docker compose -f docker-compose.prediction-stats.yml up -d
```

#### Локальная разработка

```bash
# Однократный расчет
pnpm run predictions:stats:calc

# Постоянная работа
pnpm run predictions:stats:calc:loop
```

### 🔍 Что было исправлено

#### Проблема

На проде коллекция `predictionStats` была пустая, потому что:

1. Не было автоматического расчета статистики
2. Расчет нужно было запускать вручную
3. Не было интеграции с импортом матчей

#### Решение

1. Создан воркер для автоматического расчета
2. Добавлен loop режим в скрипт
3. Воркер запускается через Docker Compose
4. Работает параллельно с импортом матчей

### 📝 Миграция

Если у вас уже запущены старые воркеры:

```bash
# 1. Остановить старые воркеры
docker compose -f docker-compose.matches-import-forward.yml down

# 2. Пересобрать с новым Dockerfile
docker compose -f docker-compose.matches-import-forward.yml build

# 3. Запустить заново
docker compose -f docker-compose.matches-import-forward.yml up -d

# 4. Запустить новый воркер статистики
docker compose -f docker-compose.prediction-stats.yml up -d
```
