# 📋 Резюме реализации профиля пользователя со статистикой

## Что было добавлено

### 1. Компоненты React

#### `src/components/profile/PredictionStatsCard.tsx`

- Компонент для отображения агрегированной статистики прогнозов
- Показывает: всего событий, выиграло, проиграло, hit rate, ROI, не определено
- Включает визуализацию с прогресс-баром
- Использует shadcn/ui компоненты (Card, Badge)

#### `src/components/profile/UserPredictionsList.tsx`

- Компонент для отображения списка прогнозов пользователя
- Поддерживает сортировку: новые/старые/лучшие/худшие
- Показывает статус каждого прогноза (рассчитан/в ожидании)
- Отображает статистику каждого прогноза (выиграло/всего, hit rate)
- Ссылки на полный текст прогноза

### 2. API маршруты

#### `src/app/api/predictions/stats/[userId]/route.ts`

- GET маршрут для получения агрегированной статистики
- Параметр: `userId` (path parameter)
- Возвращает: total, won, lost, undecided, hitRate, roi
- Также возвращает метаданные: количество прогнозов, рассчитанных, в ожидании

#### `src/app/api/predictions/user/[userId]/route.ts`

- GET маршрут для получения списка прогнозов пользователя
- Параметры: `userId` (path), `sort` (query: recent|oldest|best|worst)
- Возвращает: массив прогнозов с их статистикой
- Поддерживает сортировку по дате и качеству

### 3. Обновления страницы профиля

#### `src/app/(frontend)/(site)/profile/page.tsx`

- Добавлены импорты новых компонентов
- Добавлены вкладки (Tabs) для переключения между разделами

- Вкладка "Мои прогнозы" - отображает `UserPredictionsList`
- Добавлен компонент `PredictionStatsContainer` для загрузки статистики

## Функциональность

### Вкладка "Общая статистика"

Отображает:

- ✅ Всего событий (сумма по всем прогнозам)
- ✅ Выиграло (количество и процент)
- ✅ Проиграло (количество и процент)
- ✅ Hit Rate (процент точности)
- ✅ ROI (средний ROI)
- ✅ Не определено (количество и процент)
- ✅ Прогресс-бар с распределением результатов

### Вкладка "Мои прогнозы"

Функции:

- ✅ Список всех прогнозов пользователя
- ✅ Сортировка по дате (новые/старые)
- ✅ Сортировка по качеству (лучшие/худшие)
- ✅ Статус каждого прогноза (рассчитан/в ожидании)
- ✅ Статистика каждого прогноза (выиграло/всего, hit rate)
- ✅ Ссылки на полный текст прогноза
- ✅ Форматирование дат (русская локаль)

## Технические детали

### Зависимост��

Используются существующие зависимости:

- `payload` - для Local API
- `react` - для компонентов
- `next` - для маршрутов
- `shadcn/ui` - для UI компонентов
- `date-fns` - для форматирования дат

### Стили

- Tailwind CSS для стилизации
- shadcn/ui компоненты (Card, Badge, Button, Select, Tabs)
- Цветовая схема: зелёный (выигрыши), красный (проигрыши), жёлтый (неопределённые)

### Производительность

- Загрузка данных один раз при монтировании компонента
- Ленивая загрузка статистики при переходе на вкладку
- Кэширование на клиенте (можно расширить)
- Пагинация на сервере (до 1000 прогнозов)

## Как использовать

### 1. Убедитесь что статистика рассчитана

```bash
node --loader @esbuild-kit/esm-loader scripts/prediction-stats/calculate-all.mjs
```

### 2. Откройте страницу профиля

```
http://localhost:3100/profile
```

### 3. Переключайтесь между вкладками

- Вкладка "Общая статистика" - видите метрики
- Вкладка "Мои прогнозы" - видите список с сортировкой

## Интеграция с существующим кодом

### AuthContext

Компоненты используют `useAuth()` для получения текущего пользователя:

```typescript
const { user, isLoading, refreshUser } = useAuth()
```

### Payload CMS

API маршруты используют Payload Local API для получения данных:

```typescript
const payload = await getPayload({ config })
const stats = await payload.find({
  collection: 'predictionStats',
  where: { author: { equals: userId } },
})
```

### Коллекции

Используются существующие коллекции:

- `posts` - для получения прогнозов
- `predictionStats` - для получения статистики
- `users` - для получения информации о пользователе
