# 📋 Шпаргалка по созданию исходов в Outcome Groups

## 🎯 Основные типы исходов

### 1. Исходы матча (П1, Х, П2)

**П1 (Победа хозяев)**

```
Название: П1
Условие 1:
  - comparisonOperator: Равно (=)
  - outcomeValue: 1
```

**Х (Ничья)**

```
Название: Х
Условие 1:
  - comparisonOperator: Равно (=)
  - outcomeValue: 0
```

**П2 (Победа гостей)**

```
Название: П2
Условие 1:
  - comparisonOperator: Равно (=)
  - outcomeValue: 2
```

---

### 2. Двойной шанс (1Х, 12, Х2)

**1Х (Победа хозяев или ничья)**

```
Название: 1Х
Условие 1:
  - comparisonOperator: Принадлежность множеству (in)
  - set: [1, 0]
```

**12 (Победа любой команды)**

```
Название: 12
Условие 1:
  - comparisonOperator: Принадлежность множеству (in)
  - set: [1, 2]
```

**Х2 (Ничья или победа гостей)**

```
Название: Х2
Условие 1:
  - comparisonOperator: Принадлежность множеству (in)
  - set: [0, 2]
```

---

### 3. Тоталы (ТБ, ТМ)

**ТБ (Тотал больше) - с линиями**

```
Название: ТБ
Условие 1:
  - comparisonOperator: Больше (>)
  - calculationType: Сумма обеих команд (sum)
  - value: (не указывать, придёт от пользователя)
Значения для UI:
  - 0.5, 1.5, 2.5, 3.5, 4.5, 5.5
```

**ТМ (Тотал меньше) - с линиями**

```
Название: ТМ
Условие 1:
  - comparisonOperator: Меньше (<)
  - calculationType: Сумма обеих команд (sum)
  - value: (не указывать, придёт от пользователя)
Значения для UI:
  - 0.5, 1.5, 2.5, 3.5, 4.5, 5.5
```

**ТБ 2.5 (Тотал больше 2.5) - фиксированное значение**

```
Название: ТБ 2.5
Условие 1:
  - comparisonOperator: Больше (>)
  - calculationType: Сумма обеих команд (sum)
  - value: 2.5
Значения для UI: (не указывать)
```

---

### 4. Индивидуальные тоталы (ИТБ, ИТМ)

**ИТБ(1) 1.5 (Индивидуальный тотал хозяев больше 1.5)**

```
Название: ИТБ(1) 1.5
Условие 1:
  - comparisonOperator: Больше (>)
  - calculationType: Только хозяева (home)
  - value: 1.5
```

**ИТМ(2) 1.5 (Индивидуальный тотал гостей меньше 1.5)**

```
Название: ИТМ(2) 1.5
Условие 1:
  - comparisonOperator: Меньше (<)
  - calculationType: Только гости (away)
  - value: 1.5
```

---

### 5. Обе забьют (ОЗ)

**ОЗ Да (Обе команды забьют)**

```
Название: ОЗ Да
Условие 1:
  - comparisonOperator: Больше или равно (≥)
  - calculationType: Минимум обеих команд (min)
  - value: 1
```

**ОЗ Нет (Хотя бы одна не забьёт)**

```
Название: ОЗ Нет
Условие 1:
  - comparisonOperator: Меньше (<)
  - calculationType: Минимум обеих команд (min)
  - value: 1
```

---

### 6. Комбинированные исходы

**ОЗ + ТБ 2.5 (Обе забьют И тотал больше 2.5)**

```
Название: ОЗ + ТБ 2.5
Условие 1:
  - comparisonOperator: Больше или равно (≥)
  - calculationType: Минимум обеих команд (min)
  - value: 1
Условие 2:
  - comparisonOperator: Больше (>)
  - calculationType: Сумма обеих команд (sum)
  - value: 2.5
conditionLogic: И (AND)
```

**ОЗ ИЛИ ТБ 2.5 (Обе забьют ИЛИ тотал больше 2.5)**

```
Название: ОЗ ИЛИ ТБ 2.5
Условие 1:
  - comparisonOperator: Больше или равно (≥)
  - calculationType: Минимум обеих команд (min)
  - value: 1
Условие 2:
  - comparisonOperator: Больше (>)
  - calculationType: Сумма обеих команд (sum)
  - value: 2.5
conditionLogic: ИЛИ (OR)
```

---

## 🔧 Типы вычислений (calculationType)

| Тип            | Описание                  | Пример использования |
| -------------- | ------------------------- | -------------------- |
| **sum**        | Сумма обеих команд        | ТБ, ТМ (общий тотал) |
| **min**        | Минимум обеих команд      | ОЗ (обе забьют)      |
| **max**        | Максимум обеих команд     | Редко используется   |
| **home**       | Только хозяева            | ИТБ(1), ИТМ(1)       |
| **away**       | Только гости              | ИТБ(2), ИТМ(2)       |
| **difference** | Разница (хозяева - гости) | Форы                 |

## 🔍 Операторы сравнения

| Оператор    | Символ | Когда использовать          |
| ----------- | ------ | --------------------------- |
| **gt**      | >      | ТБ (тотал больше)           |
| **gte**     | ≥      | ОЗ (минимум 1 гол)          |
| **lt**      | <      | ТМ (тотал меньше)           |
| **lte**     | ≤      | Редко используется          |
| **eq**      | =      | П1, Х, П2 (точное значение) |
| **neq**     | ≠      | Редко используется          |
| **in**      | ∈      | 1Х, 12, Х2 (множест��о)     |
| **between** | []     | Диапазон значений           |
| **even**    | чёт    | Чёт/нечёт                   |
| **odd**     | нечёт  | Чёт/нечёт                   |

## 💡 Частые ошибки

### ❌ Неправильно

```
ТБ 2.5:
  - calculationType: НЕ УКАЗАН
  - value: 2.5
```

**Проблема:** Не указан способ вычисления

### ✅ Правильно

```
ТБ 2.5:
  - comparisonOperator: Больше (>)
  - calculationType: Сумма обеих команд (sum)
  - value: 2.5
```

---

### ❌ Неправильно

```
П1:
  - calculationType: sum
  - value: 1
```

**Проблема:** Для исходов матча нужен outcomeValue, а не calculationType

### ✅ Правильно

```
П1:
  - comparisonOperator: Равно (=)
  - outcomeValue: 1
```

---

### ❌ Неправильно

```
1Х:
  - outcomeValue: 1
  - outcomeValue: 0  // Нельзя указать дважды!
```

**Проблема:** Для множества значений нужен set

### ✅ Правильно

```
1Х:
  - comparisonOperator: Принадлежность множеству (in)
  - set: [1, 0]
```

---

## 🎓 Логика объединения условий

### AND (И) - все условия должны выполниться

```
ОЗ + ТБ 2.5:
  Условие 1: min(голы) ≥ 1  ✓
  Условие 2: sum(голы) > 2.5  ✓
  conditionLogic: AND

Результат: ОБА условия должны быть истинны
```

### OR (ИЛИ) - хотя бы одно условие

```
ОЗ ИЛИ ТБ 2.5:
  Условие 1: min(голы) ≥ 1  ✓
  Условие 2: sum(голы) > 2.5  ✗
  conditionLogic: OR

Результат: Достаточно ОДНОГО истинного условия
```

## 📝 Быстрый старт

### Создание простого исхода (например, П1)

1. **Создайте группу исходов** → "Основные исходы"
2. **Добавьте исход** → нажмите "Add Outcome"
3. **Заполните название** → "П1"
4. **Добавьте условие** → нажмите "Add Condition"
5. **Выберите оператор** → "Равно (=)"
6. **Заполните outcomeValue** → 1
7. **Сохраните**

### Создание исхода с тоталом (например, ТБ 2.5)

1. **Добавьте исход** → "ТБ 2.5"
2. **Добавьте условие**
3. **Выберите оператор** → "Больше (>)"
4. **Выберите calculationType** → "Сумма обеих команд (sum)"
5. **Заполните value** → 2.5
6. **Сохраните**

### Создание комбинированного исхода (например, ОЗ + ТБ 2.5)

1. **Добавьте исход** → "ОЗ + ТБ 2.5"
2. **Добавьте первое условие:**
   - Оператор: "Больше или равно (≥)"
   - calculationType: "Минимум обеих команд (min)"
   - value: 1
3. **Добавьте второе условие:**
   - Оператор: "Больше (>)"
   - calculationType: "Сумма обеих команд (sum)"
   - value: 2.5
4. **Выберите conditionLogic** → "И (AND)"
5. **Сохраните**

### ⚠️ Важно!

- **Поля появляются автоматически** в зависимости от выбранного оператора
- Если видите лишние поля - это старые данные, создайте новый исход
- Для исходов матча (П1/Х/П2) используйте **outcomeValue**
- Для тоталов (ТБ/ТМ) используйте **calculationType + value**
- Для двойного шанса (1Х/12/Х2) используйте оператор **"in" + set**

## 🔗 Связь с маркетами

Каждая группа исходов привязывается к маркету через поле `groups` в коллекции Markets.
Маркет определяет:

- `statType`: тип статистики (goals, numeric, outcome)
- `statPath`: путь к данным (например, "corners", "yellowCards")

Исход использует эти данные для вычисления результата.
