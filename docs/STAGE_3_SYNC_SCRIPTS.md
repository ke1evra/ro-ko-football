# Этап 3: Скрипты синхронизации данных

**Статус**: ✅ Завершён (2025-01-15)

**Цель**: Создать скрипты для автоматической и ручной синхронизации данных из LiveScore API.

## Сценарии синхронизации событий матчей

### 1 сценарий: Вместе с импортом матчей (встроенная логика)

- **Когда**: При импорте матчей через `import-matches-forward.mjs` или `sync-fixtures.mjs`
- **Что делает**: Автоматически скачивает события для каждого импортированного матча
- **Преимущества**: События сразу доступны после импорта матчей
- **Реализация**: Встроено в `matches-history-sync.mjs` → `fetchAndUpsertStatsForMatch`

### 2 сценарий: Отдельно, по существующим матчам ретроспективно

- **Когда**: Для матчей, у которых еще нет событий в `matchEvents`
- **Что делает**: Ищет матчи без событий и скачивает их отдельно
- **Преимущества**: Можно запускать в любое время для заполнения исторических данных
- **Реализация**: `sync-match-events-retro.mjs` - ручной скрипт

## Реализованные скрипты

### Автосинхронизация

#### 1. `sync-fixtures.mjs` - Синхронизация фикстур

**Расписание**: Каждый час
**Цель**: Обновление предстоящих матчей с коэффициентами и событиями

**Особенности**:

- Обрабатывает все активные лиги
- Синхронизирует до 1000 фикстур за раз
- Пакетная обработка по 50 матчей
- Обновляет существующие + создаёт новые фикстуры
- Встроенная история коэффициентов (обновляет oddsHistory при обновлении фикстур)
- Скачивает события матчей (если доступны) и сохраняет в matchEvents

**Запуск**:

```bash
# Ручной запуск
node scripts/sync-fixtures.mjs

# Через Docker
docker-compose -f docker-compose.sync-fixtures.yml up sync-fixtures
```

#### ~~2. `sync-odds-history.mjs` - История коэффициентов~~

**Статус**: ❌ Удалён - коэффициенты обновляются в `sync-fixtures.mjs`

**Причина**: Частое обновление коэффициентов (каждые 30 секунд) не требуется. Достаточно обновления каждый час вместе с фикстурами.

#### 3. `sync-match-events.mjs` - События матчей (live)

**Расписание**: Каждые 15 секунд (вперёд/назад/луп)
**Цель**: Синхронизация событий live матчей

**Особенности**:

- Обрабатывает только live матчи
- Создаёт/обновляет события (голы, карточки, замены)
- Нормализация типов событий
- Детальная информация об игроках

**Запуск**:

```bash
node scripts/sync-match-events.mjs
```

#### 4. `sync-match-events-retro.mjs` - Ретроспективная синхронизация событий

**Расписание**: По запросу (ручной запуск)
**Цель**: Скачивание событий для существующих матчей без событий

**Особенности**:

- Ищет матчи без связанных событий в `matchEvents`
- Скачивает события из `/scores/events.json`
- Обрабатывает завершенные и live матчи
- Полезно для заполнения исторических данных

**Запуск**:

```bash
node scripts/sync-match-events-retro.mjs
```

#### 5. `sync-seasons.mjs` - Сезоны лиг

**Расписание**: Раз в день (2:00)
**Цель**: Обновление информации о сезонах

**Особенности**:

- Синхронизирует все активные лиги
- Определяет текущий сезон
- Долгий кэш (24 часа) - данные редко меняются

**Запуск**:

```bash
node scripts/sync-seasons.mjs
```

## Архитектура скриптов

### Общая структура

```javascript
async function syncXxx() {
  // 1. Инициализация Payload
  const payload = await getPayload({ config })

  // 2. Получение данных из API через loggedFetch
  const data = await loggedFetch.get('/endpoint.json', params, {
    source: 'script',
    ttl: XXX,
  })

  // 3. Нормализация данных
  const normalized = normalizeData(data)

  // 4. Пакетная обработка
  for (const batch of createBatches(normalized)) {
    await processBatch(payload, batch)
  }

  // 5. Статистика
  console.log(`Обработано: ${processed}, Создано: ${created}, Обновлено: ${updated}`)
}
```

### Логирование и обработка ошибок

- **Подробное логирование** каждого шага
- **Graceful error handling** - продолжение работы при ошибках
- **Статистика выполнения** - created/updated/errors
- **Rate limiting** - автоматический контроль лимита 45k/день

### Оптимизации

## Troubleshooting

### Проблема: Скрипт зависает

**Симптомы**: Долго выполняется, не завершается

**Решение**:

1. Проверить подключение к БД
2. Проверить лимит API (45k/день)
3. Увеличить таймауты в скрипте
4. Разбить на меньшие пачки

### Проблема: Дублирование данных

**Симптомы**: Создаются дубликаты записей

**Решение**:

1. Проверить уникальные индексы в коллекциях
2. Проверить логику поиска существующих записей
3. Добавить дополнительные проверки на дубликаты

### Проблема: Cron jobs не запускаются

**Симптомы**: Скрипты не выполняются по расписанию

**Решение**:

1. Проверить статус Docker контейнеров
2. Проверить cron логи: `docker logs <container>`
3. Проверить синтаксис cron выражений
4. Перезапустить cron сервисы
