# Этап 2: Новые коллекции Payload

**Статус**: ✅ Завершён (2025-01-15)

**Цель**: Создать Payload коллекции для хранения нормализованных данных из LiveScore API.

## Реализованные коллекции

### 1. `Fixtures` - Фикстуры матчей

**Назначение**: Хранение запланированных матчей с коэффициентами и историей их изменений.

**Ключевые особенности**:

- **fixtureId**: Уникальный ID из LiveScore API
- **oddsHistory**: Встроенный массив с полной историей коэффициентов
- **Связь с матчем**: Когда фикстура становится матчем
- **Автосинхронизация**: Каждый час обновляется

**Структура**:

```typescript
{
  fixtureId: number,           // Уникальный ID
  date: Date,                  // Дата матча
  time: string,                // Время (HH:MM)
  homeTeam: { id, name, logo }, // Команда хозяев
  awayTeam: { id, name, logo }, // Команда гостей
  competition: { id, name },    // Соревнование
  league: relationship,         // Связь с Leagues
  status: 'scheduled' | 'live' | 'finished' | ...,
  odds: { pre: {...}, live: {...} }, // Текущие коэффициенты
  oddsHistory: [...],           // История коэффициентов
  match: relationship,          // Связь с Matches (опционально)
  lastSyncAt: Date,
  syncSource: 'fixtures' | 'live' | 'manual'
}
```

### 2. `Seasons` - Сезоны лиг

**Назначение**: Справочник сезонов для каждой лиги.

**Ключевые особенности**:

- **seasonId**: Уникальный ID сезона из API
- **isCurrent**: Флаг текущего активного сезона
- **Автосинхронизация**: Раз в день

**Структура**:

```typescript
{
  seasonId: number,     // ID сезона из API
  name: string,         // Название (2024/25, 2024)
  startDate: Date,      // Дата начала
  endDate: Date,        // Дата окончания
  isCurrent: boolean,   // Текущий сезон?
  league: relationship, // Связь с Leagues
  lastSyncAt: Date
}
```

### 3. `Standings` - Турнирные таблицы

**Назначение**: Хранение позиций команд в турнирных таблицах.

**Ключевые особенности**:

- **Ручная синхронизация**: Запускается администратором
- **Полная статистика**: Голы, победы, поражения, очки
- **Связь с сезоном**: Для правильной группировки

**Структура**:

```typescript
{
  league: relationship,     // Лига
  season: relationship,     // Сезон
  position: number,         // Позиция в таблице
  team: { id, name },       // Команда
  played: number,           // Сыграно матчей
  won: number, drawn: number, lost: number, // Статистика
  goalsFor: number, goalsAgainst: number,   // Голы
  points: number,           // Очки
  form: string,             // Форма (WWDLL)
  lastSyncAt: Date
}
```

### 4. `MatchEvents` - События матчей

**Назначение**: Хранение событий матчей (голы, карточки, замены).

**Ключевые особенности**:

- **Автосинхронизация**: Вперед/назад/луп (регулярные о��новления)
- **Детальная информация**: Игрок, ассистент, тип события
- **Связь с матчем**: Для группировки событий

**Структура**:

```typescript
{
  match: relationship,      // Матч
  minute: number,           // Минута события
  type: 'goal' | 'yellow' | 'red' | 'substitution' | ...,
  team: 'home' | 'away',    // Команда
  player: { id?, name? },   // Игрок
  assistPlayer?: string,    // Ассистент
  playerOut?: string,       // Ушёл с поля (замена)
  playerIn?: string,        // Вышел на поле (замена)
  description?: string,     // Описание
  lastSyncAt: Date
}
```

### 5. `Teams` - Футбольные команды

**Назначение**: Справочник команд с логотипами и информацией.

**Ключевые особенности**:

- **teamId**: Уникальный ID команды из API
- **Ручная синхронизация**: Запускается администратором
- **Логотипы и стадионы**: Визуальная информация

**Структура**:

```typescript
{
  teamId: number,           // ID команды из API
  name: string,             // Название команды
  country: { id, name },    // Страна
  logo?: string,            // URL логотипа
  stadium?: string,         // Домашний стадион
  lastSyncAt: Date
}
```

### 6. `Countries` - Страны

**Назначение**: Справочник стран для команд и лиг.

**Ключевые особенности**:

- **countryId**: Уникальный ID страны из API
- **FIFA коды**: Для международных стандартов
- **Ручная синхронизация**: Запускается администратором

**Структура**:

```typescript
{
  countryId: number,        // ID страны из API
  name: string,             // Название страны
  flag?: string,            // URL флага
  fifaCode?: string,        // FIFA код (RUS, GER)
  lastSyncAt: Date
}
```

## Стратегия синхронизации

### Автосинхронизация (фоновые процессы):

- **Fixtures**: Каждый час (основной источн��к данных)
- **Seasons**: Раз в день (справочные данные)
- **MatchEvents**: Вперед/назад/луп (регулярные обновления)

### Ручная синхронизация (через админку):

- **Standings**: По запросу (тяжёлые данные)
- **Teams**: По запросу (справочные данные)
- **Countries**: По запросу (редко меняются)

## Безопасность и доступ

### Права доступа:

- **Чтение**: Доступно всем пользователям (публичные данные)
- **Создание/Изменение/Удаление**: Только через синхронизацию (программно)

### ApiRequestLogs:

- **Чтение**: Только администраторы
- **Создание**: Только через код логирования

## Troubleshooting

### Проблема: Коллекц��и не создались

**Симптомы**: Коллекции не видны в админ-панели

**Решение**:

1. Проверить `payload.config.ts` - все коллекции импортированы и добавлены
2. Перезапустить Payload: `pnpm run payload generate:types`
3. Проверить логи на ошибки миграции БД

### Проблема: Связи не работают

**Симптомы**: Relationship поля пустые

**Решение**:

1. Проверить `relationTo` в конфигурации полей
2. Убедиться, что связанные коллекции существуют
3. Проверить данные в связанных коллекциях

### Проблема: Индексы не создались

**Симптомы**: Поиск работает медленно

**Решение**:

1. Проверить конфигурацию индексов в коллекциях
2. Перезапустить Payload для применения индексов
3. Проверить логи MongoDB на ошибки создания индексов
