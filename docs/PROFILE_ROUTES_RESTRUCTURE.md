# 📋 Реструктуризация маршрутов профиля

## Обзор

Маршруты профиля пользователя были реструктурированы для разделения личного профиля (редактирование) и открытого профиля (просмотр).

---

## Структура маршрутов

### Старая структура ❌

```
/profile              → Редактирование профиля (требует авторизации)
/users/[username]     → Открытый профиль пользователя
```

### Новая структура ✅

```
/my-profile           → Редактирование профиля текущего пользователя (требует авторизации)
/profile/[username]   → Открытый профиль пользователя (доступен всем)
```

---

## Маршруты

### 1. `/my-profile` - Личный профиль (редактирование)

**Файл:** `src/app/(frontend)/(site)/my-profile/page.tsx`

**Тип:** Клиентский компонент (`'use client'`)

**Требует авторизацию:** ✅ Да

**Функции:**

- Редактирование профиля (имя, username, биография)
- Загрузка аватара
- Редактирование социальных сетей (сайт, Twitter, GitHub)
- Просмотр статистики прогнозов
- Просмотр списка своих прогнозов

**Доступ:**

```
GET /my-profile
```

**Пример использования:**

```tsx
import Link from 'next/link'

export function ProfileLink() {
  return <Link href="/my-profile">Мой профиль</Link>
}
```

---

### 2. `/profile/[username]` - Открытый профиль (просмотр)

**Файл:** `src/app/(frontend)/(site)/profile/[username]/page.tsx`

**Тип:** Серверный компонент

**Требует авторизацию:** ❌ Нет (доступен всем)

**Функции:**

- Просмотр информации о пользователе
- Просмотр биографии
- Просмотр рейтинга
- Просмотр статистики прогнозов
- Просмотр постов пользователя
- Просмотр комментариев пользователя
- Ссылки на социальные сети

**Доступ:**

```
GET /profile/[username]
```

**Примеры:**

```
/profile/ivan          → Профиль пользователя ivan
/profile/john_doe      → Профиль пользователя john_doe
```

**Пример использования:**

```tsx
import Link from 'next/link'

export function UserProfileLink({ username }: { username: string }) {
  return <Link href={`/profile/${username}`}>Профиль</Link>
}
```

## Миграция ссылок

### Обновление ссылок в компонентах

**Было:**

```tsx
// Редактирование профиля
<Link href="/profile">Мой профиль</Link>

// Открытый профиль
<Link href={`/users/${username}`}>Профиль пользователя</Link>
```

**Стало:**

```tsx
// Редактирование профиля
<Link href="/my-profile">Мой профиль</Link>

// Открытый профиль
<Link href={`/profile/${username}`}>Профиль пользователя</Link>
```

### Обновленные компоненты

1. **PredictorCard** (`src/components/predictions/PredictorCard.tsx`)
   - Обновлена ссылка на полный профиль: `/profile/${author.username}`

## Функциональность

### `/my-profile` - Редактирование

**Форма редактирования:**

- Имя
- Username
- Биография
- Аватар (загрузка файла или URL)
- Социальные сети:
  - Сайт
  - X/Twitter
  - GitHub

**Статистика:**

- Общая статистика прогнозов (Hit Rate, ROI, всего, выиграло)
- Список своих прогнозов

**Сохранение:**

- Кнопка "Сохранить изменения"
- Сообщение об успехе/ошибке

---

### `/profile/[username]` - Просмотр

**Информация о пользователе:**

- Аватар
- Имя и username
- Рейтинг (сумма очков)
- Биография
- Ссылки на социальные сети

**Статистика прогнозов:**

- Таблица с результатами каждого прогноза
- Показатели: события, выиграло, проиграло, hit-rate, ROI, очки

**Активность:**

- Список постов пользователя
- Список комментариев пользователя

**Дизайн:**

- Адаптивный макет (мобильный/десктоп)
- Карточки для каждого элемента
- Цветовая индикация (зелёный для выигрышей, красный для проигрышей)

## Примеры

### Переход на свой профиль

```tsx
import Link from 'next/link'

export function MyProfileButton() {
  return (
    <Link href="/my-profile">
      <button>Редактировать профиль</button>
    </Link>
  )
}
```

### Переход на профиль другого пользователя

```tsx
import Link from 'next/link'

export function UserProfileLink({ user }: { user: User }) {
  return (
    <Link href={`/profile/${user.username}`}>
      <div className="flex items-center gap-2">
        <img src={user.avatar} alt={user.name} />
        <span>{user.name}</span>
      </div>
    </Link>
  )
}
```

### Из компонента PredictorCard

```tsx
// Ссылка на полный профиль прогнозиста
<Link href={`/profile/${author.username}`}>
  <Button>Полный профиль</Button>
</Link>
```

## Редирект

Если нужно перенаправить старые ссылки на новые, можно добавить редирект в `next.config.mjs`:

```javascript
// next.config.mjs
export default {
  async redirects() {
    return [
      {
        source: '/profile',
        destination: '/my-profile',
        permanent: true,
      },
      {
        source: '/users/:username',
        destination: '/profile/:username',
        permanent: true,
      },
    ]
  },
}
```

## Тестирование

### Проверка маршрутов

1. **Личный профиль (требует авторизацию)**

   ```
   GET /my-profile
   ```

   - Если не авторизован → показать "Требуется авторизация"
   - Если авторизован → показать форму редактирования

2. **Открытый профиль (доступен всем)**

   ```
   GET /profile/ivan
   GET /profile/john_doe
   ```

   - Если пользователь существует → показать профиль
   - Если пользователь не существует → показать "Пользователь не найден"

### Проверка ссылок

- [ ] Ссылка "Мой профиль" ведёт на `/my-profile`
- [ ] Ссылка на профиль прогнозиста ведёт на `/profile/[username]`
- [ ] Кнопка "Полный профиль" в PredictorCard ведёт на `/profile/[username]`
