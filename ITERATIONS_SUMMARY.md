## Общие принципы исправлений

### TypeScript в Node.js
- Для запуска TypeScript файлов используется `tsx` (TypeScript executor)
- `tsx` автоматически компилирует TypeScript в памяти
- Позволяет импортировать `.ts` файлы напрямую

### Импорты в ESM модулях
- При использовании `tsx` импорты должны использовать расширение `.ts`
- При использовании `node` с `.js` файлами импорты должны использовать расширение `.js`
- Для пакетов Next.js требуется явное ��казание расширения `.js` (например, `'next/server.js'`)

### Проверка переменных окружения
- Переменные окружения должны проверяться **перед** инициализацией Payload
- Используется загрузка из нескольких источников для гибкости
- Выводятся понятные сообщения об ошибках при отсутствии переменных

### Логирование и диагностика
- Выводится информация о загруженных переменных окружения
- Чувствительные данные маскируются для безопасности
- Логируются этапы инициализации для отладки
- Используются префиксы `[INIT]`, `[STEP]`, `[SYNC]` для структурирования логов
